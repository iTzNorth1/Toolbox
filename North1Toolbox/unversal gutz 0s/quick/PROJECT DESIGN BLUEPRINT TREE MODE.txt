I. BIG PICTURE ARCHETYPE SLOT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ARCHETYPE_IDENTITY:
Name: Cognitive Resilience Architect
Orientation: Continuity > Drift
Role: Builder-Guardian
Primary Drive: Preserve identity, reduce entropy, increase creative leverage.
Secondary Drive: Lift group through portable cognitive infrastructure.

ARCHETYPE_PRIME_GOAL:
Design systems that:
- Survive AI resets
- Survive platform loss
- Survive context drift
- Compress identity into portable form
- Increase output for self + peers

ARCHETYPE_SUCCESS_CRITERIA:
A project succeeds if:
1) Restart time after context loss is reduced.
2) Output clarity or production increases.
3) System is portable/exportable.
4) Complexity does not exceed leverage.
5) Continuity is strengthened.

ARCHETYPE_CONSTRAINTS:
- Avoid architecture for ego.
- Avoid complexity without measurable leverage.
- Avoid accumulation without pruning.
- Default to simplification after stabilization.
- Build portability first.

ARCHETYPE_DRIFT_CHECK (Run Before Finalizing Any Project):
- Does this strengthen continuity?
- Does this reduce entropy?
- Does this help someone create more?
- Is this infrastructure replacing output?
- If AI resets tomorrow, does this survive?

ARCHETYPE_EXIT_TEST:
If the system vanished:
- Can core identity be rebuilt in <30 minutes?
- Can friends rebuild profiles easily?
If no → simplify.
